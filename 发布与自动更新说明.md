# 梦想小镇脚本发布与即时更新

## 最终结构

- Tampermonkey 中安装的是稳定的本地加载桥；加载桥含本机路径，仅保存在本地，不纳入 Git。
- 加载桥保持旧 v3.77 的 `@name` 与 `@namespace`，因此沿用原脚本的 GM 存储、面板配置和长期调度状态。
- 加载桥通过本地 `@require`，在每次游戏页面启动时读取：
  `<LOCAL_RELEASE_DIR>\梦想小镇日常一体化.user.js`
- GitHub 仓库仍是正式版本源、提交历史和远程备份；主脚本继续维护 `@updateURL`、`@downloadURL` 与 `.meta.js`。

## 固定发布流程

1. 在 `<REPOSITORY_DIR>` 修改主脚本并递增版本号。
2. 运行语法检查、完整回归和 `git diff --check`。
3. 仅在全部检查通过后，将主脚本覆盖到上述本地固定发布路径。
4. 核对仓库主脚本与本地发布文件的 SHA-256 完全一致。
5. 刷新真实游戏页面，确认面板载入新版本、旧配置保留、Scheduler 正常运行。
6. 同步 `.meta.js`，提交并推送 GitHub；确认本地 `HEAD` 与 `origin/main` 一致且工作区干净。
7. 将实测结果和提交哈希写入本机 handoff 文档；该文件不提交到公开仓库。

## 验证命令

```powershell
node --check '梦想小镇日常一体化.user.js'
python 'tests\regression_checks.py'
git diff --check
```

本地发布文件还应单独运行完整回归，并与仓库文件比较 SHA-256。

## 重要约束

- 不要把开发中或尚未通过测试的文件覆盖到固定发布路径。
- 日常功能修改只更新主脚本，不修改或重新安装加载桥。
- 如果未来新增 `@grant`、修改 `@match`、改变运行站点或移动发布文件路径，必须同步修改加载桥，并安排一次人工重新安装与存储继承验证。
- Tampermonkey 必须开启“允许访问文件网址”，且网站访问权限设为“在所有网站上”；否则控制台会出现 `Access to this or all local files is forbidden`。
- GitHub 推送不可省略。本地加载桥解决及时生效，GitHub负责正式发布与恢复。
- `梦想小镇日常本地加载器.user.js` 必须保持为本地忽略文件，不得添加到 Git。
